# 课程知识库与来源策略

课程大纲只定义“教什么”，不能直接充当课程正文。弦上乐理的正文、练习解释和规则校验按以下链路生产：

```text
检索可信来源
→ 拆分为可验证的知识断言
→ 建立知识节点与来源映射
→ 编写适合吉他的原创中文讲解
→ 规则引擎校验音名、音程、音阶与和弦
→ 人工审核
→ 发布课程与练习
```

## 来源优先级

1. 开放教材和大学教学资源，例如 Open Music Theory、OpenStax、UNSW Music Acoustics；
2. 有明确教学责任主体的课程资料，例如 Berklee Online；
3. 乐器厂商的基础吉他技术资料，例如 Fender；
4. 其他资料只用于发现关键词或交叉核对，不作为单一事实依据。

论坛、无作者聚合页和模型生成文本不能作为核心乐理事实的唯一来源。

## 内容版权

- 知识库保存事实的原创中文转述，不复制受版权保护页面的段落、图表或音频；
- CC BY / CC BY-SA 来源记录许可证并保留署名链接；
- 商业网站仅作事实核验和课程顺序参考；
- 歌曲案例使用原创、公版或已授权材料。

## 数据位置

- `src/data/knowledge.ts`：来源登记、知识断言、课程映射、审核状态；
- `src/data/catalog.ts`：面向学习者的原创课程与练习；
- `src/theory/index.ts`：不依赖网络或大模型的确定性乐理规则；
- `scripts/validate-content.mjs`：内容数量、依赖关系、来源完整性和乐理事实校验。

每个发布课程必须至少映射一个 `reviewed` 知识节点；每个知识断言至少有一个可访问的来源 URL。涉及易混淆事实或物理声学时，优先要求两个独立来源交叉核验。
